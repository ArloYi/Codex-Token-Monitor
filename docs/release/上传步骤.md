# Codex Token Monitor 发布清单

## 源码仓库

项目根目录就是 Git 仓库根目录。公开发布前确认：

1. `zsh ./scripts/test.sh` 通过。
2. `zsh ./scripts/privacy-check.sh` 通过。
3. `git status --short --ignored` 中没有准备提交的 `build/`、`dist/`、`releases/`、`.DS_Store`、数据库、日志或本机 Codex 状态。
4. README、`LICENSE`、`DISCLAIMER.md`、`PRIVACY.md` 与 `SECURITY.md` 内容一致。
5. 新功能通过独立分支和 Pull Request 合并，不直接推送默认分支。

## Release 附件

运行：

```bash
zsh ./scripts/package-release.sh
```

脚本会在 `dist/` 生成当前版本的 Universal 2 ZIP 和 SHA-256 文件。本机归档可放在 `releases/v<版本>/`，`dist/` 与 `releases/` 都不提交到 Git。

创建 GitHub Release 时：

1. 标签与 `App/Info.plist` 中的版本一致。
2. Target 选择已通过 CI 的默认分支提交。
3. 上传 ZIP 与对应 `.sha256` 文件。
4. 说明用户可见变化、最低 macOS 版本、辅助功能权限用途、未公证限制、非商业许可和风险自担条款。
5. 附件上传完成并核对 SHA-256 后再公开 Release。

## 不得公开的内容

- `~/.codex` 下的状态、SQLite、会话索引、rollout、提示词、回复和日志。
- Token、密码、Cookie、私钥、账号资料及其他凭据。
- 本机绝对路径、真实项目名称、私有用量数据和未经许可的第三方素材。
- `build/`、`dist/`、`releases/`、`.DS_Store` 和临时调试文件。
