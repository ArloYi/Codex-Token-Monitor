# Codex 额度浮窗 PRD

> 本文保留原作者单行横向浮窗的设计记录。当前本地重整版已改为“顶部额度条 + 悬停向下展开三行详情”，实际行为以根目录的 `使用说明.md` 和 `README.zh-CN.md` 为准。

## AI 速读卡

- 产品一句话：只在 Codex 前台出现的 macOS 浮窗，持续展示额度和当前项目 Token。
- 核心循环：打开 Codex → 查看额度 → 切换项目 → 项目名称与统计自动更新。
- 目标平台：macOS 13+，原生 AppKit，依赖本机 Codex 桌面应用。
- 硬约束：不抢焦点、不占菜单栏、不上传数据、不修改 Codex、不安装自启动任务。
- 推荐默认：右下角磨砂单行浮窗，项目 5 秒刷新，额度 60 秒刷新。
- 发挥空间：材质、间距、把手细节、错误文案和更稳健的数据兼容层。
- P0 验收：额度、项目本周、全项目本周、占比和历史总量可信显示。
- 最容易翻车：读取错误的项目状态字段，或刷新时重新置顶导致抢焦点。
- 超预期机会：数据口径说明、兼容性诊断、隐私清单和可复现的安全测试。

## 第一章：产品概述

Codex 额度浮窗是一款 macOS 本地状态工具，让高频使用 Codex 的用户能够持续查看额度与项目 Token，而无需反复打开额度入口。

### 1.1 差异化对比表

以下对比基于用户提供的 Codex 使用现状与本项目已实现行为，未对所有第三方工具进行市场核验。

| 功能 | 现有方式 | 本产品 | 实现方式 |
|---|---|---|---|
| 查看额度 | 进入 Codex 内部入口 | Codex 前台时持续可见 | 本机 `app-server` 读取额度窗口 |
| 项目归因 | 需要自行判断 | 跟随当前 `selected-project` | 状态文件与 SQLite 聚合 |
| 菜单栏占用 | 常见小工具占菜单栏 | 不占菜单栏 | `LSUIElement` 浮窗 |
| 焦点安全 | 外置窗口可能抢前台 | 切到其他应用立即隐藏 | 不可激活 `NSPanel` 与 Workspace 通知 |
| 信息密度 | 指标分散 | 一行表达项目值、分母和占比 | 单行公式与自适应字号 |

### 1.2 三类用户画像

| 角色 | 核心目标 | 最大不满 | 切换触发点 |
|---|---|---|---|
| Codex 重度用户 | 随时知道额度还能否支撑当前任务 | 每次进入入口查询打断工作 | 额度始终在视线边缘且不抢焦点 |
| 多项目开发者 | 判断哪个项目消耗最多 | 项目名称和 Token 缺少绑定 | 切换项目后 5 秒内自动更新 |
| 成本敏感的独立开发者 | 建立本周和历史消耗感知 | 单个数字缺少分母和占比 | 同时显示项目值、本周总值和占比 |

### 1.3 可行性边界

| 在范围内（及原因） | 明确排除在外（及原因） |
|---|---|
| macOS 本机浮窗，AppKit 可提供不可激活窗口 | Windows、Linux 和移动端，当前实现依赖 AppKit |
| 读取用户本机 Codex 状态 | 修改 Codex 安装包或注入界面，升级风险与安全风险过高 |
| 本机项目 Token 统计 | 将统计定义为官方账单，二者口径不完全等价 |
| 手动启动应用 | 自动登录启动，当前不应增加系统常驻配置 |
| 源码构建和本地临时签名 | Apple 公证与自动更新，需要开发者证书和发布基础设施 |

### 1.4 约束分层

| 硬约束 | 推荐默认 | 发挥空间 |
|---|---|---|
| 不调用应用激活、强制置顶或请求注意力 API | 右下角、38px 高、磨砂材质 | 圆角、透明度和阴影 |
| 仅 Codex 前台时显示 | 项目 5 秒、额度 60 秒刷新 | 兼容更多 Codex 状态版本 |
| 主体点击穿透 | 左侧移动、右侧缩放 | 把手图形和悬停反馈 |
| 数据不上传 | 9.5pt 至 12pt 自适应字号 | 更清晰的错误与恢复提示 |
| 当前项目以 `selected-project` 为准 | 无效时回退 `active-workspace-roots` | 诊断快照与统计解释 |

## 第三章：核心模块详细设计

### 第 3.1 节 单行额度与 Token 摘要

#### a) ASCII 图

```text
(额度剩余 60%) ｜ (演示工具项目本周消耗 1200万 / 12000万 · 占比 10.0%) ｜ (总消耗 Token 1.00亿)
```

加载态：`额度与 Token 读取中`
额度失败：`额度读取中 ｜ 演示工具项目本周消耗 1200万 / 12000万 · 占比 10.0% ｜ 总消耗 Token 读取中`
项目失败：`额度剩余 60% ｜ 项目读取中 ｜ 总消耗 Token 1.00亿`

#### b) 交互流程

```text
定时刷新 --> 获得额度与项目 --> 格式化单位 --> 仅更新 label.stringValue
    +-- 额度接口失败 --> 保留上次有效额度，继续显示项目数据
    +-- 项目状态失败 --> 保留上次有效项目，下一周期重试
```

#### c) 状态清单

| 名称 | 触发条件 | 视觉标识 | 退出条件 |
|---|---|---|---|
| 初始读取 | 尚无快照 | “读取中” | 任一数据源返回 |
| 完整 | 两类数据均有效 | 三段完整文本 | 数据失效或应用退出 |
| 部分可用 | 仅额度或项目有效 | 对缺失段显示“读取中” | 下一次成功刷新 |
| 缓存兜底 | 历史总量远端为空 | 使用上次有效值 | 远端返回更可信值 |

#### d) 依赖关系

```text
Codex app-server ---> QuotaSnapshot ----+
                                        +--> FloatingSummaryText --> NSTextField
Codex state/sqlite --> ProjectSnapshot -+
```

只读本机数据；窗口位置和宽度写入 `NSUserDefaults`，不写 Codex 状态。

#### e) 待决问题

1. 此处未解决：Codex 后续是否稳定保留当前 `app-server` 方法与响应字段。
2. 此处未解决：官方额度窗口与自然周的产品文案是否需要进一步区分。

### 第 3.2 节 当前项目识别与本周聚合

#### a) ASCII 图

```text
selected-project: 演示工具
        |
        v
/Users/user/Documents/SampleProject
        |
        +--> 项目本周 1200万
        +--> 所有项目本周 12000万
        +--> 占比 10.0%
```

#### b) 交互流程

```text
读取 .codex-global-state.json
    |
    +-- selected-project 有效 --> 使用对应 rootPaths 的首个路径
    +-- selected-project 无效 --> 回退 active-workspace-roots
    +-- 两者都无效 --> 保留旧快照并在 5 秒后重试
    |
    v
查询 state_5.sqlite --> 必要时读取 rollout --> 聚合项目值与总值
    +-- SQLite 不存在 --> 返回 0，等待后续恢复
    +-- rollout 不可读 --> 使用数据库累计值
```

#### c) 状态清单

| 名称 | 触发条件 | 视觉标识 | 退出条件 |
|---|---|---|---|
| 已选择项目 | `selected-project` 可解析 | 显示项目全称 | 用户切换项目 |
| 回退项目 | 当前选择字段缺失 | 显示 active root 项目 | 当前字段恢复 |
| 零消耗 | 项目窗口内没有 Token | `0.00万` 与 `0.0%` | 产生新消耗 |
| 数据暂缺 | 状态或数据库无法解析 | 项目读取中或旧值 | 下一次成功刷新 |

#### d) 依赖关系

读取 `~/.codex/.codex-global-state.json`、`~/.codex/state_5.sqlite` 和数据库记录引用的 rollout 文件。数据流为单向只读。

#### e) 待决问题

1. 此处未解决：远程项目和多根目录项目的归因规则。
2. 此处未解决：被删除项目的历史 Token 是否进入独立“未归类”统计。

### 第 3.3 节 焦点安全与可见性

#### a) ASCII 图

```text
Codex 前台    (浮窗可见，非激活，主体穿透)
微信前台      (浮窗隐藏)
Chrome 前台   (浮窗隐藏)
数据刷新      (只改文字，不 orderFront)
```

#### b) 交互流程

```text
NSWorkspaceDidActivateApplicationNotification
    |
    +-- bundle id = com.openai.codex --> 定位并 orderFront
    +-- 其他 bundle id --> orderOut
    +-- 无法识别前台应用 --> 安全默认隐藏
```

失败路径一：刷新回调晚于应用切换，回调只更新文字，不重新显示。
失败路径二：浮窗接收把手鼠标事件，窗口仍使用 `NonactivatingPanel`，不得成为键盘焦点。

#### c) 状态清单

| 名称 | 触发条件 | 视觉标识 | 退出条件 |
|---|---|---|---|
| Codex 可见 | Codex 前台 | 浮窗出现 | 其他应用激活 |
| 后台隐藏 | 非 Codex 前台 | 无浮窗 | Codex 再次激活 |
| 拖动中 | 左侧把手按下 | 闭合手形光标 | 松开鼠标 |
| 缩放中 | 右侧把手按下 | 左右缩放光标 | 松开鼠标 |

#### d) 依赖关系

依赖 AppKit `NSPanel`、`NSWorkspace` 和 `NSEvent`。不依赖辅助功能权限、通知权限或屏幕录制权限。

#### e) 待决问题

无。焦点安全优先级高于浮窗持续可见。

## 第四章：超越竞品的差异化功能

### 第 4.1 节 焦点安全契约

1. 竞品为何没有这个功能：基于公开工具形态的推断，菜单栏工具通常不需要处理“只在宿主前台显示”；普通浮窗则容易把持续可见当成最高优先级。
2. 本产品如何实现：把不可激活、主体穿透、后台隐藏和刷新不置顶写入代码与静态回归测试。
3. 交互流程：

```text
使用 Codex --> 浮窗提供信息 --> 切换微信 --> 浮窗让位 --> 回到 Codex --> 恢复显示
```

4. 风险与应对：Codex bundle id 或窗口机制变化时，安全默认隐藏，禁止退化为全局置顶。

### 第 4.2 节 项目消耗公式化表达

1. 竞品为何没有这个功能：此处未验证。结构上需要同时连接当前项目状态、历史线程和额度窗口，数据源比单一额度百分比复杂。
2. 本产品如何实现：把“项目值 / 本周总值 · 占比”放在同一视觉分组，历史总量独立使用亿。
3. 交互流程：

```text
切换示例工具乙 --> 识别项目 --> 聚合项目 Token --> 与本周总值计算占比 --> 一行同步更新
```

4. 风险与应对：本地历史不完整时明确其不是官方账单，避免把估算包装为计费事实。

### 第 4.3 节 超预期机会

- 兼容性诊断卡：一键生成不含路径和内容的字段可用性报告。
- 口径悬停说明：在不增加常驻噪声的前提下解释分子、分母和回退窗口。
- 版本迁移守卫：检测 Codex 状态结构变化后安全隐藏项目统计，而不是显示错误项目。

## 第五章：数据模型

```jsonc
{
  "version": 1, // 必填；快照结构版本
  "quota": { // 额度快照
    "usedPercent": 40.0, // 必填；已用百分比，0...100
    "remainingPercent": 60.0, // 必填；剩余百分比，0...100
    "resetsAt": 1784476800, // 可空；额度窗口重置 Unix 时间
    "windowDurationMinutes": 10080, // 默认值: 10080；统计窗口分钟数
    "lifetimeTokens": 100000000 // 默认值: 0；历史总 Token
  },
  "project": { // 当前项目统计
    "name": "演示工具", // 必填；用户可见项目全称
    "path": "/Users/user/Documents/SampleProject", // 必填；本机项目根目录，不对外输出
    "weeklyProjectTokens": 12000000, // 默认值: 0；项目窗口内 Token
    "weeklyTotalTokens": 120000000, // 默认值: 0；所有项目窗口内 Token
    "sharePercent": 10.0 // 默认值: 0；项目值除以总值
  },
  "window": { // 浮窗偏好
    "originX": 620, // 默认值: 当前屏幕右侧 20px；窗口左下角 X
    "originY": 20, // 默认值: 当前屏幕底部 20px；窗口左下角 Y
    "width": 760, // 默认值: 760；允许 560...屏幕可用宽度
    "fontSize": 12.0 // 派生值；允许 9.5...12.0
  }
}
```

额度和项目快照分离，因为两者刷新频率不同。窗口偏好只保存几何信息，不保存账号、项目内容或凭证。展示文本是派生结果，不持久化。

## 第六章：技术架构

```text
+---------------- AppKit 表现层 ----------------+
| NSPanel 可见性、材质、把手、字体测量          |
+---------------- 领域格式层 ------------------+
| Token 单位、项目占比、单行摘要                |
+---------------- 本地数据层 ------------------+
| Codex app-server | global state | SQLite/rollout |
+---------------- 安全守卫层 ------------------+
| 非激活 API 扫描、签名、Info.plist、回归测试   |
```

| 库名 | 用途 | 为何优于替代方案 | 大致包体积 |
|---|---|---|---|
| AppKit | 原生浮窗和工作区通知 | 系统自带，无第三方窗口层 | 系统框架，不进入应用包 |
| Foundation | JSON、进程、文件和时间 | 系统自带，与 Objective-C 直接集成 | 系统框架，不进入应用包 |
| sqlite3 CLI | 查询 Codex 本地线程 | macOS 自带，当前无需链接额外库 | 系统组件，不进入应用包 |

最大架构风险是依赖非公开稳定承诺的 Codex 本地状态结构。应把字段解析集中在 `CodexDataSource`，字段缺失时保留旧值或显示读取态，不把解析失败扩散到窗口生命周期。

可替换技术原则：推荐维持 Objective-C + AppKit；若未来迁移 Swift，可替换语言实现。不可变的是数据只读、窗口不可激活、其他应用前台时隐藏、刷新不重新置顶和同一统计口径。

## 第七章：交互细节

### 7.1 键盘快捷键

当前无键盘快捷键。应用不获取键盘焦点，增加快捷键会与核心安全约束冲突。

### 7.2 右键菜单与上下文菜单

当前无右键菜单。主体点击穿透，仅两端把手接收鼠标事件。

```text
左侧六点：移动
右侧双线：调整长度
其他区域：事件传给 Codex
```

### 7.3 空状态

| 视图 | 用户所见 | CTA |
|---|---|---|
| 初次读取 | 额度与 Token 读取中 | 无，自动重试 |
| 无项目 | 项目读取中 | 在 Codex 中选择本地项目 |
| 无历史 | 项目本周消耗 0.00万 | 正常使用 Codex 后自动更新 |

### 7.4 错误状态

| 触发条件 | 用户提示 | 恢复操作 |
|---|---|---|
| Codex CLI 不存在 | 额度读取中 | 安装或更新 Codex |
| 额度接口无响应 | 保留上次额度 | 60 秒后自动重试 |
| 状态 JSON 暂时不可读 | 保留上次项目 | 5 秒后自动重试 |
| SQLite 不存在 | 本周显示 0 | 运行一次 Codex 任务 |
| 保存位置不在任一屏幕 | 无错误弹窗 | 自动回到当前屏幕右下角 |

### 7.5 加载状态

首次读取立即显示文字加载态，不使用动画。后台刷新小于 200ms 时不显示状态变化；超过 200ms 继续保留旧值；超过一个刷新周期仍失败时显示对应分段的“读取中”仅适用于没有旧值的情况。

### 8.3 批量处理流程

本产品没有批量用户任务。发布输出流程为：

```text
运行测试
   |
   +-- 失败 --> 停止打包并返回错误
   |
   v
构建应用 --> 临时签名 --> ZIP 打包 --> 输出 dist 路径
```

构建、签名和打包必须串行，避免产物与源码版本不一致。

## 第九章：开发优先级

### P0

- 正确读取额度、当前项目、本周统计和历史总量。
- 仅 Codex 前台可见，切换其他应用立即隐藏。
- 不抢焦点，刷新不重新置顶。
- 项目切换后 5 秒内更新。

### P1

- 可移动、可调整长度、字体自适应并保存几何信息。
- 数据失败时保留旧值并自动恢复。
- 构建、测试、隐私和使用说明完整。

### P2

- 兼容性诊断卡与结构变化检测。
- 可下载的 GitHub Release、校验和与更友好的首次安装说明。
- 真实界面截图和版本化变更记录。

### P3

- Apple Developer ID 签名、公证和自动更新。
- 多账户或远程项目口径，前提是 Codex 提供稳定数据边界。

## 第十一章：开发者交接说明

### a) 实现顺序建议

你应先保证 `CodexDataSource` 的字段解析和统计口径，再实现单行格式，最后接入窗口生命周期。数据正确但界面简陋仍可验证；窗口漂亮但项目归因错误会直接破坏信任。完成数据后，依次验证焦点安全、几何交互和发布打包。

### b) 最可能导致返工的三个决策

1. 当前项目来源
   安全默认：优先 `selected-project`。
   改变信号：Codex 明确废弃该字段，且出现新的稳定当前项目标识。
2. 浮窗显示策略
   安全默认：仅 Codex 前台显示。
   改变信号：用户明确选择全局常驻模式，并接受遮挡与焦点风险。
3. 本周定义
   安全默认：使用额度窗口，缺失时最近 7 天。
   改变信号：官方提供自然周或账单周期的稳定接口。

### c) 哪里要严格，哪里可以灵活

- 第一章与第五章：约束，产品定位和数据语义不能自行改写。
- 第二章与第三章：约束，单行、动态项目与安全行为构成核心体验。
- 第四章：建议，可以替换超预期机会，但不得扩大 P0。
- 第六章：建议，可以迁移 Swift，但必须保留安全和数据不变量。
- 第七章：约束，不能为了更多交互让主体获取焦点。
- 第八至十章：建议，可以优化工具链和指标，但必须可测量。

### d) 已知的未知项

- 此处未解决：Codex 本地状态和 `app-server` 接口的长期兼容承诺。
- 此处未解决：Apple 公证、自动更新和发布证书由谁维护。
- 此处未解决：远程项目、多根目录项目和被删除项目的归因口径。
- 此处未解决：账户历史总 Token 与本机累计差异的官方解释。

### e) 验收剧本

验收剧本 1：在 macOS 13+ 执行 `zsh ./scripts/test.sh`，应看到 `all tests passed`，并用退出码 0 验证。
验收剧本 2：打开示例工具甲，再点击示例工具乙，浮窗应在 5 秒内从“示例工具甲项目”变为“示例工具乙项目”，并用屏幕录制验证。
验收剧本 3：保持浮窗可见后切换到微信、飞书或 Chrome，浮窗应在 100ms 目标内隐藏，前台应用不应跳回 Codex，并用屏幕录制验证。
验收剧本 4：拖动左侧六点并缩放右侧双线，重启应用后应恢复位置和宽度，字体保持 9.5pt 至 12pt 且单行不截断。
验收剧本 5：执行 `zsh ./scripts/package-release.sh`，应生成带版本号的 ZIP；解压后 `codesign --verify --deep --strict` 应通过，并确认包内没有 `~/.codex` 数据。
